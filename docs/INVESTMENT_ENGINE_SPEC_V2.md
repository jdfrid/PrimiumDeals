### 2.2 Alpaca – המלצה ראשית למסחר אוטומטי

**יתרונות:**
- **עמלות $0** על מניות ו־ETF
- **API-first** – בנוי למפתחים
- **Paper Trading חינמי** – בדיקה לפני כסף אמיתי
- **נתוני שוק** – Real-time בחינם (תוכניות בסיס)
- **תמיכה בישראל** – חשבון לישראלים (W8-BEN)
- **SPY, QQQ, VOO** – מדדים מובילים זמינים
- **מניות חלקיות** – מסחר מ־$1

**הפקדה:**
- Wire (SWIFT) מבנק ישראלי → המרה ל־USD
- עלות העברה: לפי הבנק (בד"כ 50–150 ש"ח)
- אין עמלה מצד Alpaca על הפקדה

**חסרונות:**
- אין כרטיס אשראי ישיר – רק Wire
- אין מסחר ישיר בשקלים

### 2.3 eToro – אלטרנטיבה להפקדה קלה

**יתרונות:**
- **כרטיס אשראי** – הפקדה מיידית
- **פופולרי בישראל** – תמיכה בעברית
- **API מלא** – ביצוע אוטומטי
- מינימום נמוך (~$50)

**חסרונות:**
- Spread גבוה יותר ממסחר "קלאסי"
- API פחות מפותח מ־Alpaca
- פחות מתאים ל־algo trading מתקדם

### 2.4 Interactive Brokers – למתקדמים

**יתרונות:**
- API מקצועי (TWS/IB Gateway)
- גישה ל־135+ בורסות
- עמלות נמוכות (Lite: $0)

**חסרונות:**
- דורש התקנת TWS או IB Gateway
- מורכב יותר להגדרה
- הפקדה: Wire בלבד

## 3. תדירות מסחר – פעם ביום vs מספר מחזורים

### המלצה: **פעם ביום (או פעמיים)**

**סיבות:**
1. **עלויות** – מסחר תכוף מגדיל spread ועמלות
2. **מדדים/ETF** – נזילים פחות בשעות פתיחה/סגירה
3. **אופטימלי** – מסחר באמצע היום (10:00–15:00 ET) – spread צר יותר
4. **פשטות** – קל יותר לבדוק, לתחזק ולתקן באגים
5. **אסטרטגיה** – השקעה במדדים = long-term, לא day trading

**מתי לשקול מספר מחזורים:**
- אם יש סיגנלים real-time חזקים
- אם רוצים להגיב מהר לאירועים
- אם יש backtesting שמראה יתרון

**המלצה מעשית:** התחל עם **מחזור אחד ביום** (למשל 15:00 ישראל = 8:00 ET), ואפשר להרחיב אחרי שמוכיחים שהמערכת יציבה.

## 5. מנוע כללים (Rules Engine)

### 5.1 דוגמאות לכללים

| כלל | תנאי | פעולה |
|-----|------|-------|
| **Stop-Loss** | מחיר ירד מתחת ל־20% מהרכישה | מכור את כל הפוזיציה |
| **Take-Profit** | מחיר עלה מעל 20% מהרכישה | מכור את הרווח (העודף מעל סכום ההשקעה), החזר לקופה |
| **Rebalance** | סטייה מההקצאה המתוכננת | קנה/מכור להחזיר לאיזון |
| **DCA** | תאריך קבוע (למשל 1 בחודש) | קנה סכום קבוע |

### 5.2 מבנה כללים (טכני)

```
Rule = {
  id, name, enabled,
  condition: { type, field, operator, value },  // e.g. price_change < -0.20
  action: { type, params },                      // e.g. sell_all, sell_profit
  asset_filter: [ "SPY", "QQQ" ]                // על אילו נכסים
}
```

### 5.3 ממשק ניהול

- פאנל ניהול (Backend + Admin UI)
- הוספה/עריכה/מחיקה של כללים
- הפעלה/כיבוי
- לוג ביצוע – מתי כל כלל הופעל ומה בוצע
